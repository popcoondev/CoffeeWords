# Firebase Admin SDK用サービスアカウントキーの取得方法

Node.jsスクリプト `create-test-data.js` を実行する場合、Firebase Admin SDKの認証が必要です。
以下の手順でサービスアカウントキーを取得し、設定してください。

## サービスアカウントキーの取得

1. Firebase Consoleにログイン: https://console.firebase.google.com/
2. プロジェクトを選択
3. 左側のメニューから「プロジェクトの設定」を選択
4. 「サービスアカウント」タブをクリック
5. 「新しい秘密鍵の生成」ボタンをクリック
6. JSONファイルがダウンロードされます

## 環境変数の設定

ダウンロードしたJSONファイルを安全な場所に保存し、以下の方法で環境変数に設定します：

### Linux/macOS

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/your-project-firebase-adminsdk-xxxxx-xxxxxxxxxx.json"
```

### Windows (コマンドプロンプト)

```cmd
set GOOGLE_APPLICATION_CREDENTIALS=C:\path\to\your-project-firebase-adminsdk-xxxxx-xxxxxxxxxx.json
```

### Windows (PowerShell)

```powershell
$env:GOOGLE_APPLICATION_CREDENTIALS="C:\path\to\your-project-firebase-adminsdk-xxxxx-xxxxxxxxxx.json"
```

## 注意事項

- このサービスアカウントキーは秘密情報です。Gitリポジトリにコミットしないでください。
- `.gitignore` に `*.json` などのパターンを追加することをお勧めします。
- 本番環境では、環境変数を使用してサービスアカウント認証情報を管理してください。

## 代替手段

サービスアカウントキーを使用したくない場合は、以下の方法があります：

1. `create-test-data-simple.js` スクリプトを使用する（ブラウザから実行可能）
2. `create-admin-user.js` スクリプトを使用してユーザーを作成し、そのユーザーでログインしてアプリからデータを作成

これらの方法はAdmin SDKを必要としないため、サービスアカウントキーの設定が不要です。
